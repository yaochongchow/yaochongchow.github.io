Software / Backend Services

Backend Services (API)

Correct APIs under concurrency, rate limits, and live traffic.

Challenges

Fan-out amplifies everything. A race condition invisible at low traffic becomes a consistency bug at scale, and one slow handler poisons tail latency for the whole request.

Approach

Clean REST/gRPC contracts, stateless handlers, and request flows that are actually traceable. Not just "works in staging."

Projects

Global rate limiters, commerce checkout workflows, live telemetry ingestion paths.

Tools

Go Node.js Python REST gRPC WebSockets

Software / Data & Runtime

Data & Runtime (DB)

Caching, queues, and storage that start changing how the system behaves.

Challenges

Hot keys, queue buildup, and write contention don't show up in unit tests. They show up when caches, queues, persistence, and containers are all interacting under real load.

Approach

Thinking through schema boundaries and cache strategy before they become production fires, not after.

Projects

Redis Cluster with Lua scripting for rate limiting, queue-backed coordination layers, relational storage workflows.

Tools

PostgreSQL Redis DynamoDB MySQL Kafka Docker Kubernetes

Software / Observability & Ops

Observability & Ops (OBS)

Metrics, logs, and traces that make production debugging feel less like archaeology.

Challenges

p95 spikes, flapping targets, retry storms. Hard to catch if observability was bolted on after the fact. By then the signal is already noisy.

Approach

Dashboards and alerts designed around incident questions, not just "green means good."

Projects

Prometheus/Grafana service health setups, CloudWatch operational views, distributed system dashboards for live debugging.

Tools

AWS CloudWatch Prometheus Grafana CI/CD Linux Git

Software / Distributed Coordination

Distributed Coordination (SYS)

Retries, ordering, and stale state across service boundaries.

Challenges

Racing requests, hidden partial failures, state that looks consistent but isn't. Clean code doesn't save you. You have to reason about timeouts, ordering, and where your isolation boundaries actually sit.

Approach

Designing around backpressure, idempotency, and event ordering from the start rather than patching it in later.

Projects

Global rate limiting under concurrent load, queue-driven commerce flows, real-time telemetry pipelines.

Tools

Distributed Systems Microservices API Design Real-time Streaming Performance Tuning

Robotics Systems (ROS)

Navigation and control loops that stay debuggable once the robot is moving.

Projects

Robotics / Embedded & Device Debugging

Embedded & Device Debugging (FW)

When the problem is near the hardware and abstractions stop helping.

Challenges

Intermittent device behavior and BLE/serial issues are nearly impossible to debug without visibility at the boundary. Higher layers hide the source fast.

Approach

Getting signal through logs, interfaces, and firmware-adjacent instrumentation before the problem propagates upward.

Projects

BLE bring-up instrumentation, boundary debugging using RTT and UART logging, device-state tracing on nRF52840 hardware.

Tools

C/C++ BLE RTOS UART RTT On-device Instrumentation

Projects
